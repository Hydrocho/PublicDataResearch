Preprocessing

Colab 전처리 체크리스트

전처리는 분석의 기본 자료를 정리하고, 통계적 검증은 분석 결과의 의미를 판단하는 중요한 단계입니다.

전처리 항목 확인 내용 예시
인코딩 확인 한글이 깨지지 않도록 파일 인코딩을 확인합니다. utf-8-sig, cp949
결측치 확인 비어 있는 값이 많은 열이나 행을 확인합니다. 주소 없음, 기관명 없음
중복값 제거 같은 기관이나 지역이 반복 입력된 경우 정리합니다. 동일 기관 중복 등록
지역명 통일 서로 다른 지역 표기 방식을 하나로 맞춥니다. 서울특별시 → 서울
데이터 병합 여러 데이터를 공통 기준으로 연결합니다. 시도, 시군구, 법정동코드
파생 변수 생성 정책 판단에 필요한 새로운 지표를 만듭니다. 노인 1만 명당 돌봄기관 수

정책과 앱 제안으로 연결하기

분석 결과는 표와 그래프로 끝나지 않고, 실제 문제 해결을 위한 실행 방안으로 바뀌어야 합니다.

정책 제안: 돌봄 취약지역 우선 지원

지역별 돌봄 취약지수를 기준으로 우선 지원 지역을 선정하고, 방문형 돌봄 인력과 이동형 서비스, 지역별 예산 배분을 강화합니다.

취약지수 우선 지원 예산 배분 방문 돌봄

앱 제안: 지역 돌봄 매칭 서비스

주민과 보호자가 가까운 돌봄기관, 이용 가능한 복지 서비스, 신청 방법을 쉽게 찾을 수 있도록 지도 기반 서비스를 제공합니다.

핵심 메시지

공공데이터는 사회 문제를 객관적으로 바라보게 해 주고, NotebookLM은 데이터에서 탐구 가능한 질문과 분석 관점을 찾고 전처리 코드 초안을 작성하도록 도와주며, Google Colab은 그 코드를 실행·수정하면서 데이터를 직접 분석하고 시각화할 수 있게 해 줍니다. 이 과정을 통해 우리는 데이터에 근거한 정책과 앱 서비스를 제안할 수 있습니다.
